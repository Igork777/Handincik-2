Adults Web API Documentation

Adult

The Adult is sent back and forth as json.
The json format is seen in the example below:
{
"JobTitle": "Tutor",
"ID": 0,
"FirstName": "Aryan",
"LastName": "Gifford",
"HairColor": "Black",
"EyeColor": "Brown",
"Age": 56,
"Weight": 99.2,
"Height": 172,
"Sex": "M"
}

API Request Description Exceptions
Get
The request will return an array of Adults
Example: .../Adults 		No exception can occur
Post The request will add a new user to the system if such doesn't exist
Example: .../Adults/{json Adult object} 		Internal exception 500 can occur if such adult already exists
Delete The request delete a user if such user exists
Example: .../Adults/?firstname={firstname}&lastname={lastname} 		Internal exception 500 if no such adult can be deleted

User

The User is sen back and forth as json. User is used for loggining in system.

API Request Description Exceptions
Get The request will return the corresponding User object based on the inputted username and password
Example: ...Users/?username={username}&password={password}		 Internal exception 500 if there is no user with corresponding username and password
Post The request will add new User, specifying firstname and lastname
Example: ...Users/{json object User} 		Internal exception 500 if there is a user with such credentials already exist
